## Codebase Patterns

- **Cookie name:** auth usa `accessToken` (n√£o `access_token`) ‚Äî setCookie em auth.controller.ts line ~89
- **Client components:** NUNCA importar `@/lib/auth` (barrel) em `use client` ‚Äî importar direto de `@/lib/auth/auth-provider`
- **PhoneInput:** usa react-international-phone + usePhoneInput hook; forwardRef expondo ref do input nativo; `forceDialCode: true` mant√©m Brasil selecionado
- **looksLikePhone:** `const clean = v.replace(/[^\d]/g, ""); return clean.length >= 8` ‚Äî funciona com E.164 e nacional
- **normalizePhone backend:** `phone.replace(/\D/g, "")` strip tudo ‚Äî aceita qualquer formato
- **UAZAPI_INSTANCE_TOKEN env:** fallback quando nenhuma connection no DB tem uazapiToken
- **igniter schema:** regenerar manualmente quando CLI falha (ajv/dist/core error)
- **apiFetch helper:** retorna response.json() raw ‚Äî data.data √© o array
- **Git mmap workaround:** Use `git hash-object -w <file>` then `git update-index --add --cacheinfo` when `git add` fails. For commits: rebuild index with `git ls-tree -r HEAD`, then `git write-tree` + `git commit-tree` + `git update-ref`
- **Prisma migrate shadow DB issues:** When `prisma migrate dev` fails with shadow DB errors, create migration SQL manually and use `prisma migrate deploy`. Mark existing migrations as applied with `prisma migrate resolve --applied <name>`
- **Prisma v7 datasource:** URL goes in `prisma.config.ts`, NOT in schema.prisma `datasource` block. Remove `url = env("DATABASE_URL")` from schema
- **Corrupted git objects:** `.cursor/rules` tree object `fdc1aea` is missing from packfile. Workaround: skip it during index rebuild, use `git ls-tree -r HEAD 2>&1 | grep -v "^error:"`
- **dev server:** roda em http://localhost:3000 ‚Äî verificar com `curl -s http://localhost:3000/api/health`
- **Tailwind dark mode / CSS vars:** globals.css define vars como `oklch(...)` diretamente (N√ÉO como valores de canal HSL). Ao usar inline styles, usar `var(--popover)` etc. diretamente ‚Äî NUNCA `hsl(var(--popover))` pois quebra o color. Em classes Tailwind usar `bg-popover`, `text-popover-foreground` etc.
- **FieldSeparator bg:** usa `bg-card` (n√£o `bg-background`) pois o separador fica dentro de um Card ‚Äî em dark mode card e background t√™m cores diferentes
- **WCAG 2.1 AA:** contraste m√≠nimo 4.5:1 texto normal, 3:1 elementos UI; touch target m√≠nimo 44√ó44px; focus-visible em todos elementos interativos
- **Shadcn components path:** componentes ficam em `src/client/components/ui/` ‚Äî usar esses, n√£o criar novos
- **Igniter data pattern:** `const { data } = api.x.y.useQuery()` ‚Äî data pode ser `{ data: [...] }` ou array direto dependendo do controller
- **Array guard pattern:** `Array.isArray(data?.data) ? data.data : Array.isArray(data) ? data : []` ‚Äî previne `.filter is not a function`
- **npm install:** always use `--legacy-peer-deps` flag due to peer dependency conflicts
- **crypto.ts AES-256-GCM:** `encrypt(plaintext)` / `decrypt(encrypted)` in `src/lib/crypto.ts` ‚Äî reuse for any secret storage
- **otpauth validate:** `totp.validate({ token, window })` returns `null` if invalid, integer delta if valid
- **otpauth Secret from stored:** `OTPAuth.Secret.fromBase32(base32)` to reconstruct from DB, NOT `new OTPAuth.Secret()`
- **Prisma $transaction array:** `db.$transaction([op1, op2])` for atomic multi-table updates
- **verifyPassword reuse:** `verifyPassword(plain, hash)` from `@/lib/auth/bcrypt` wraps `bcrypt.compare` ‚Äî use for any hash comparison (passwords, recovery codes, etc.)
- **Token issuance flow:** signAccessToken ‚Üí db.refreshToken.create ‚Üí signRefreshToken ‚Üí update ‚Üí setAuthCookies ‚Üí registerDeviceSession ‚Äî duplicated in login/totpChallenge/totpRecovery
- **2FA status check (frontend):** `/api/v1/auth/me` does NOT return `twoFactorEnabled` ‚Äî use `/api/v1/auth/totp/devices` and check for verified devices instead
- **Igniter extra body fields:** procedure-consumed fields (like `cf-turnstile-response`) aren't in Zod schema ‚Äî use `as any` cast on body object in typed `api.*.mutate()` calls
- **apiFetch helper pattern:** wrap `fetch()` with `{ credentials: 'include', headers: { 'Content-Type': 'application/json' } }` + error extraction from response body
