## Codebase patterns (reference before implementing)

- **Auth modules:** `src/server/core/auth/` (backend), `src/client/components/auth/` (frontend v3 already done)
- **AI agents:** `src/server/ai-module/ai-agents/` has CRUD + runtime + 6 builtin tools (transfer_to_human, pause_session, notify_team, schedule_callback, create_lead, search_contacts) — **Builder reuses 100%**
- **Communication:** `src/server/communication/` has instances (QR + share link), campaigns, templates — reuse
- **Igniter pattern:** controller → actions (query/mutation) → handler with `authProcedure`
- **Prisma schema:** `prisma/schema.prisma` (migrations need founder approval)
- **Cookie name:** auth uses `accessToken` (not `access_token`)
- **Client components:** NEVER import `@/lib/auth` (barrel) in `use client` — use `@/lib/auth/auth-provider` direct
- **Design tokens:** `quayer-ds-v3.html` is source of truth; tokens currently scoped in `[data-auth-v3="true"]` → US-017 migrates to `:root` global
- **Testing pipeline (Release 1):** static, unit backend, unit react, api integration, contract, e2e playwright 3-projects
- **Feature flags:** pattern established in Release 3 (`docs/auth/FEATURE_FLAGS.md`)
