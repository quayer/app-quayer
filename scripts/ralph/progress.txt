## Codebase Patterns

- **Cookie name:** auth usa `accessToken` (n├гo `access_token`) тАФ setCookie em auth.controller.ts line ~89
- **Client components:** NUNCA importar `@/lib/auth` (barrel) em `use client` тАФ importar direto de `@/lib/auth/auth-provider`
- **PhoneInput:** usa react-international-phone + usePhoneInput hook; forwardRef expondo ref do input nativo; `forceDialCode: true` mant├йm Brasil selecionado
- **looksLikePhone:** `const clean = v.replace(/[^\d]/g, ""); return clean.length >= 8` тАФ funciona com E.164 e nacional
- **normalizePhone backend:** `phone.replace(/\D/g, "")` strip tudo тАФ aceita qualquer formato
- **UAZAPI_INSTANCE_TOKEN env:** fallback quando nenhuma connection no DB tem uazapiToken
- **igniter schema:** regenerar manualmente quando CLI falha (ajv/dist/core error)
- **apiFetch helper:** retorna response.json() raw тАФ data.data ├й o array
- **Git mmap workaround:** Use `git hash-object -w <file>` then `git update-index --add --cacheinfo` when `git add` fails. For commits: rebuild index with `git ls-tree -r HEAD`, then `git write-tree` + `git commit-tree` + `git update-ref`
- **Prisma migrate shadow DB issues:** When `prisma migrate dev` fails with shadow DB errors, create migration SQL manually and use `prisma migrate deploy`. Mark existing migrations as applied with `prisma migrate resolve --applied <name>`
- **Prisma v7 datasource:** URL goes in `prisma.config.ts`, NOT in schema.prisma `datasource` block. Remove `url = env("DATABASE_URL")` from schema
- **Corrupted git objects:** `.cursor/rules` tree object `fdc1aea` is missing from packfile. Workaround: skip it during index rebuild, use `git ls-tree -r HEAD 2>&1 | grep -v "^error:"`
- **dev server:** roda em http://localhost:3000 тАФ verificar com `curl -s http://localhost:3000/api/health`
- **Tailwind dark mode / CSS vars:** globals.css define vars como `oklch(...)` diretamente (N├ГO como valores de canal HSL). Ao usar inline styles, usar `var(--popover)` etc. diretamente тАФ NUNCA `hsl(var(--popover))` pois quebra o color. Em classes Tailwind usar `bg-popover`, `text-popover-foreground` etc.
- **FieldSeparator bg:** usa `bg-card` (n├гo `bg-background`) pois o separador fica dentro de um Card тАФ em dark mode card e background t├кm cores diferentes
- **WCAG 2.1 AA:** contraste m├нnimo 4.5:1 texto normal, 3:1 elementos UI; touch target m├нnimo 44├Ч44px; focus-visible em todos elementos interativos
- **Shadcn components path:** componentes ficam em `src/client/components/ui/` тАФ usar esses, n├гo criar novos
- **Igniter data pattern:** `const { data } = api.x.y.useQuery()` тАФ data pode ser `{ data: [...] }` ou array direto dependendo do controller
- **Array guard pattern:** `Array.isArray(data?.data) ? data.data : Array.isArray(data) ? data : []` тАФ previne `.filter is not a function`
- **npm install:** always use `--legacy-peer-deps` flag due to peer dependency conflicts
- **crypto.ts AES-256-GCM:** `encrypt(plaintext)` / `decrypt(encrypted)` in `src/lib/crypto.ts` тАФ reuse for any secret storage
