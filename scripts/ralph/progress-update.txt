## Codebase Patterns

- Root page cookie: app saves 'accessToken' but root checked 'auth_token' (US-001 fix)
- Auth redirect pattern: `role === 'admin' ? '/admin' : '/integracoes'` in 5+ files without onboarding check
- Token dual-storage: localStorage + cookie (systemic issue)
- Console.logs exposing tokens in google-callback (11 instances)
- `as any` casts in 30+ locations, especially modal props in admin/integracoes
- Dead buttons (18+) across ferramentas, integracoes, contatos pages
- Fake saves in /integracoes/settings (toast without API call)
- Fake validation in /(public)/connect/[token] (hardcoded responses)
- Admin pages use native confirm()/alert() instead of shadcn AlertDialog
- Sidebar: src/client/components/app-sidebar.tsx
- Admin server actions: src/app/admin/actions.ts
- Conversations use SSE with no reconnect backoff
- onKeyPress (deprecated) used in chat components
- Padrão Igniter.js: controller → actions (query/mutation) → handler com authProcedure
- Schema Prisma: src/prisma/schema.prisma

- Admin pages pattern: 'use client', apiFetch helper, header with SidebarTrigger+Breadcrumb, Card/Table layout, Dialog for create, AlertDialog for destructive actions
- Sidebar menu items added in app-sidebar.tsx under adminMenu array
