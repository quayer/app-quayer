## Codebase Patterns

- **Cookie name:** auth usa `accessToken` (não `access_token`) — setCookie em auth.controller.ts line ~89
- **igniter schema:** regenerar com `npx igniter generate` quando controller adicionado ao router mas schema não atualizado
- **Client components:** NUNCA importar `@/lib/auth` (barrel) em `use client` — importar direto de `@/lib/auth/auth-context` ou hook específico
- **apiFetch helper:** retorna `response.json()` raw — `data.data` é o array, não `data` diretamente
- **Git mmap workaround:** Use `git hash-object -w <file>` then `git update-index --add --cacheinfo` when `git add` fails
- **After Prisma schema changes:** Always run `npx prisma generate` to regenerate client types
- **Auth controller pattern:** Token generation uses `signAccessToken` + `db.refreshToken.create` + `signRefreshToken` + `setAuthCookies(response, accessToken, refreshToken)`
- **Phone OTP flow:** Uses `VerificationCode` model with `type: WHATSAPP_OTP` and `email` field stores normalized phone number
- **uazapiClient.sendText:** `(instanceId, token, { number, text })` — endpoint POST /send/text; header `token` identifies instance
- **UAZAPI_INSTANCE_TOKEN env:** fallback when no DB connection has uazapiToken set
- **libphonenumber-js:** já instalado — usar `AsYouType` para formatar; `normalizePhone` no backend remove formatação
- **PhoneInput forwardRef:** usa React.forwardRef expondo ref do input nativo; shadcn Input NÃO tem forwardRef — usar `<input>` diretamente
