## Codebase Patterns

- migrate.js usa pg diretamente (não prisma CLI) — evita dependência do effect/fast-check
- **Git mmap workaround:** Use `git hash-object -w <file>` then `git update-index --add --cacheinfo 100644,<hash>,<path>` to stage files when `git add` fails with mmap error
- **After Prisma schema changes:** Always run `npx prisma generate` to regenerate client types before typechecking
- **Auth controller pattern:** Token generation uses `signAccessToken` + `db.refreshToken.create` + `signRefreshToken` + `setAuthCookies(response, accessToken, refreshToken)` — copy from `verifyLoginOTP` or `passkeyLoginVerify`
- **Passkey challenge type:** `PasskeyChallenge.type` is a plain `String`, supports custom values like `'conditional'` in addition to `'registration'`/`'authentication'`
- **Phone OTP flow:** Uses `VerificationCode` model with `type: 'WHATSAPP_OTP'` and `email` field stores normalized phone number
- **uazapiClient.sendText:** `(instanceId, token, { number, text })` — instanceId unused in URL; endpoint is POST /send/text; header `token` identifies instance
- **UAZAPI_INSTANCE_TOKEN env:** fallback when no DB connection has uazapiToken set
- **libphonenumber-js:** já instalado — usar `AsYouType` para formatar; `normalizePhone` no backend remove formatação via replace(/\D/g, '')
- **PhoneInput forwardRef:** usa React.forwardRef para expor ref do input interno; Input shadcn já tem forwardRef
- **Igniter.js error handling:** usar `response.badRequest(message)` para retornar 400 com mensagem amigável; wrap sendWhatsAppOTP em try/catch
